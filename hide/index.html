Transactions

Before you begin, you should have the AlgoSigner Google Chrome extension installed, as this demo makes extensive use of it. You should also make a couple of wallets on Algorand TestNet and use the TestNet ALGO Faucet to generate Algos for them, as these exercises demonstrate how to move assets between two or more addresses.

Atomic Transfer

Smart Contracts

In order to run the Smart Contract portion of this demo, you need a PureStake API Key. Get your personal API Key from PureStake and add it to the .env-EXAMPLE file in the root folder, then rename .env-EXAMPLE to .env and start the demo again.

Limit Order

A Limit Order is a Smart Contract that only pays out funds when a limit amount is reached in the contract logic.

Hash Time Lock Contract

A Hash Time Lock Contract is a Smart Contract that only pays out funds when a certain "preimage" is supplied to it. Think of a preimage as a password or passphrase you need to unlock the funds.

Split Contract

A Split Contract is a Smart Contract that can be used to disburse a payment amount to two wallets in defined proportions.
